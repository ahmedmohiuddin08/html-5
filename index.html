HTML5 Elements

HTML5 introduces most descriptive HTML elements. This makes tags easy to read , and SEO friendly. And these includes.

  • article
  • aside
  • details
  • figcaption
  • figure
  • mark
  • section
  • main
  • header
  • footer
  • nav
  • video
  • audio

A semantic element clearly describes its meaning to both the browser and the developer.

article

The article element specifies independent, self-contained content. An example of where article element can be used:

  • Forum Post
  • Blog Post
  • Newspaper article

aside

The element defines some content aside from the content it is placed, such as sidebar

figure and figcaption

figure: The figure tag sepcifies self-contained content, like illustration, diagrams, photos, code listings, etc.

figcaption: The figcaption tag defines a caption for a figure element. The figcaption element can be placed as the first or last child of the figure element.

section

The section element defines section in a document. We will find section elements containing article elements, and article elements containing section elements.

main

The main element represent the dominant content of the bod of a document. The document must not have more than one main element.

header

The header element represents a container for introductory content or a set of navigational links.

footer

The footer tag defines a footer for a document or section.

A footer tag typically contains:

  • authorship information
  • copyright information
  • contact information
  • social media links
  • back to top links
  • related documents

You can have several footer element in one document.

nav

This tag is used to defined to showcase a set of navigation links.

video

The tag is used to embed video content in a document, such as a movie clip or other video streams. One or more source tag can be used in one video tag. The browser will choose the first source it supports.

audio

The audio tag is used to embed sound content in a document, such as music or other audio streams. One or more source tag can be used inside one audio tag. The browser will choose the first source it supports.
